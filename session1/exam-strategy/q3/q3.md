# Personally Identified

A company is collecting data that is generated by its users for analysis by using an Amazon S3 data lake. Some of the data being collected and stored in Amazon S3 includes personally identifiable information (PII).

The company wants a data engineer to design an automated solution to identify new and existing data that needs PII to be masked before analysis is performed. Additionally, the data engineer must provide an overview of the data that is identified. The task of masking the data will be handled by an application already created in the AWS account. The data engineer needs to design a solution that can invoke this application in real time when PII is found.

**Which solution will meet these requirements with the LEAST operational overhead?**

- A) Create an AWS Lambda function to analyze data for PII. Configure notification settings on the S3 bucket to invoke the Lambda function when a new object is uploaded.
- B) Configure notification settings on the S3 bucket. Configure an Amazon EventBridge rule for the default event bus for new object uploads. Set the masking application as the target for the rule.
- C) Enable Amazon Macie in the AWS account. Create an Amazon EventBridge rule for the default event bus for Macie findings. Set the masking application as the target for the rule.
- D) Enable Amazon Macie in the AWS account. Create an AWS Lambda function to run on a schedule to poll Macie findings and invoke the masking application.

## Answer

**C) Enable Amazon Macie in the AWS account. Create an Amazon EventBridge rule for the default event bus for Macie findings. Set the masking application as the target for the rule.**

Amazon Macie is purpose-built for discovering and protecting PII in S3. Using EventBridge to react to Macie findings in real time and invoke the masking application provides the least operational overhead compared to building custom PII detection logic or polling.

## Why the other options are incorrect

- **A) Create an AWS Lambda function to analyze data for PII. Configure notification settings on the S3 bucket to invoke the Lambda function when a new object is uploaded.** — Building custom PII detection logic in Lambda requires significant development and maintenance effort. It also doesn't scan existing data, only new uploads, and doesn't provide the overview/reporting that Macie offers natively.
- **B) Configure notification settings on the S3 bucket. Configure an Amazon EventBridge rule for the default event bus for new object uploads. Set the masking application as the target for the rule.** — This triggers the masking application on every upload regardless of whether PII is present. It doesn't identify or classify PII — it blindly invokes masking, which doesn't meet the requirement to identify data that needs masking.
- **D) Enable Amazon Macie in the AWS account. Create an AWS Lambda function to run on a schedule to poll Macie findings and invoke the masking application.** — Polling on a schedule adds latency and doesn't meet the real-time requirement. It also introduces additional operational overhead (managing the Lambda schedule, handling missed polls) compared to the event-driven EventBridge approach.

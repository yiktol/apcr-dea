# NFS Files

A company has data in an on-premises NFS file share. The company plans to migrate to AWS. The company uses the data for data analysis. The company has written AWS Lambda functions to analyze the data. The company wants to continue to use NFS for the file system that Lambda accesses. The data must be shared across all concurrently running Lambda functions.

**Which solution should the company use for this data migration?**

- A) Migrate the data into the local storage for each Lambda function. Use the local storage for data access.
- B) Migrate the data to Amazon Elastic Block Store (Amazon EBS) volumes. Access the EBS volumes from the Lambda functions.
- C) Migrate the data to Amazon DynamoDB. Ensure the Lambda functions have permissions to access the table.
- D) Migrate the data to Amazon Elastic File System (Amazon EFS). Configure the Lambda functions to mount the file system.

## Answer

**D) Migrate the data to Amazon Elastic File System (Amazon EFS). Configure the Lambda functions to mount the file system.**

Amazon EFS provides NFS-compatible shared file storage that can be mounted by multiple Lambda functions concurrently, meeting both the NFS requirement and the shared access requirement.

## Why the other options are incorrect

- **A) Migrate the data into the local storage for each Lambda function. Use the local storage for data access.** — Lambda local storage (/tmp) is ephemeral, limited in size (up to 10 GB), and not shared across concurrent function invocations. Each invocation gets its own isolated storage, so data cannot be shared.
- **B) Migrate the data to Amazon Elastic Block Store (Amazon EBS) volumes. Access the EBS volumes from the Lambda functions.** — EBS volumes can only be attached to EC2 instances, not Lambda functions. Lambda does not support mounting EBS volumes.
- **C) Migrate the data to Amazon DynamoDB. Ensure the Lambda functions have permissions to access the table.** — DynamoDB is a key-value/document database, not a file system. It doesn't provide NFS compatibility, which is a stated requirement. Migrating file-based data to DynamoDB would require significant application refactoring.

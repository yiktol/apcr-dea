# Networking

A data engineer at a large e-commerce company is designing a new data processing pipeline within AWS. The pipeline will ingest sensitive customer data from various sources, process it using EC2 instances in a VPC, and then store the results in Amazon S3. The data engineer needs to ensure that the data remains secure throughout the pipeline and that the EC2 instances cannot be directly accessed from the internet.

**Which combination of VPC security features should the data engineer implement to meet these requirements?**

- A) Use public subnets for EC2 instances and rely solely on security groups to control access
- B) Place EC2 instances in private subnets, use a NAT gateway for internet access, and implement security groups
- C) Use VPC endpoints for S3 access and place EC2 instances in public subnets with restrictive NACLs
- D) Implement VPC peering between multiple VPCs, each containing a different part of the pipeline

## Answer

**B) Place EC2 instances in private subnets, use a NAT gateway for internet access, and implement security groups**

Private subnets prevent direct internet access to EC2 instances, the NAT gateway allows outbound connectivity when needed, and security groups provide instance-level traffic control — together meeting the security requirements.

## Why the other options are incorrect

- **A) Use public subnets for EC2 instances and rely solely on security groups to control access** — Public subnets assign public IP addresses to instances, making them directly reachable from the internet. This violates the requirement that EC2 instances cannot be directly accessed from the internet. Security groups alone are not sufficient as a defense-in-depth strategy.
- **C) Use VPC endpoints for S3 access and place EC2 instances in public subnets with restrictive NACLs** — While VPC endpoints for S3 are a good practice, placing instances in public subnets still exposes them to the internet. NACLs are stateless and harder to manage, and this still doesn't meet the requirement of preventing direct internet access.
- **D) Implement VPC peering between multiple VPCs, each containing a different part of the pipeline** — VPC peering enables communication between VPCs but doesn't address the core security requirements of preventing internet access to EC2 instances or securing the data pipeline. It adds unnecessary complexity without solving the stated problem.

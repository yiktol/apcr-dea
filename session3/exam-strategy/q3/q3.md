# Glue Workflow Orchestration

A transportation company uses AWS Glue ETL jobs to run transformations on data lake data. A data engineer must visualize dependencies between transformations and run multiple transformations concurrently to reduce processing time.

How can the data engineer meet these requirements with the LEAST operational overhead?

- A) Use a scheduled trigger to start an AWS Glue workflow that will launch the required AWS Glue jobs.
- B) Use Amazon EventBridge to schedule an event to initiate an AWS Lambda function that will launch the required AWS Glue jobs.
- C) Use Amazon EventBridge to schedule an event to initiate an AWS Step Functions workflow that will launch the required AWS Glue jobs.
- D) Launch an Amazon EC2 instance that runs Apache Airflow to start the required AWS Glue jobs.

## Answer

**A**

- **A) Use a scheduled trigger to start an AWS Glue workflow that will launch the required AWS Glue jobs.**
  - Glue workflows natively visualize job dependencies
  - Built-in concurrent execution of independent jobs
  - Scheduled triggers are fully managed — no extra services
  - Least operational overhead — single service solution

## Why the other options are incorrect

- **B) Use Amazon EventBridge to schedule an event to initiate an AWS Lambda function that will launch the required AWS Glue jobs.**
  - Lambda can start Glue jobs but no dependency visualization
  - Must build custom orchestration logic in Lambda
  - Additional service to manage (EventBridge + Lambda)
  - More overhead than native Glue workflows
- **C) Use Amazon EventBridge to schedule an event to initiate an AWS Step Functions workflow that will launch the required AWS Glue jobs.**
  - Step Functions can orchestrate but adds complexity
  - Two additional services (EventBridge + Step Functions)
  - Glue workflows already provide this natively
  - Over-engineered for Glue-to-Glue orchestration
- **D) Launch an Amazon EC2 instance that runs Apache Airflow to start the required AWS Glue jobs.**
  - EC2 + Airflow requires infrastructure management
  - OS patching, scaling, Airflow maintenance
  - Highest operational overhead of all options
  - Managed alternatives exist for this use case

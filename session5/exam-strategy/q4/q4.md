# Glue Data Catalog Encryption

A financial services company needs to implement encryption for their AWS Glue Data Catalog. They want full control over encryption keys, ability to rotate cryptographic material, modify key policies, and ensure audit logging support.

What should they implement?

- A) Customer managed keys with CloudTrail logging enabled.
- B) AWS managed keys with CloudTrail logging enabled.
- C) AWS managed keys with encryption context only.
- D) Custom encryption keys stored in AWS Secrets Manager.

## Answer

**A**

- **A) Customer managed keys with CloudTrail logging enabled.**
  - Customer managed keys (CMK) provide full control
  - Can rotate cryptographic material on schedule
  - Can modify key policies and grant access
  - CloudTrail logs all key usage for audit compliance

## Why the other options are incorrect

- **B) AWS managed keys with CloudTrail logging enabled.**
  - AWS managed keys don't allow key policy modification
  - Cannot control rotation schedule
  - Limited control over key lifecycle
  - CloudTrail logging works but key control is insufficient
- **C) AWS managed keys with encryption context only.**
  - AWS managed keys lack required control
  - Encryption context alone doesn't provide audit logging
  - Cannot rotate or modify key policies
  - Doesn't meet the full control requirement
- **D) Custom encryption keys stored in AWS Secrets Manager.**
  - Secrets Manager stores secrets, not encryption keys
  - KMS is the correct service for encryption key management
  - Cannot integrate with Glue Data Catalog encryption
  - Wrong service for this use case entirely
